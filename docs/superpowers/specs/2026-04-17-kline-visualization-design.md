# 交易记录 K 线图可视化工具设计文档

## 概述

实现 README.md 中的特性一：根据用户交易记录绘制 K 线图，标注买卖点及收益率。

## 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| Web 框架 | Streamlit | Python 数据应用首选，开发速度快，与 Plotly 天然集成 |
| 图表库 | Plotly | 交互式图表，支持缩放、悬停查看数据，生成 HTML 输出 |
| 数据处理 | Polars | 高性能 DataFrame 库，原生支持 Parquet 格式 |

## 功能需求

### 1. 数据文件加载

- 用户在界面输入框中指定 `trade.csv` 和 `prices.parquet` 的本地路径
- 加载后显示数据加载状态（成功/失败）和基本信息（记录数量、日期范围等）

### 2. 资产选择

- 展示交易记录中所有资产的列表，显示资产 ID 和名称
- 支持按资产 ID 或资产名称进行模糊过滤搜索
- 用户选择单个资产后生成该资产的图表
- 列表显示每个资产的交易次数和总收益率概览

### 3. K 线图展示

- 使用 Plotly Candlestick 图表类型
- 默认时间范围：该资产交易期间（第一笔买入 → 最后一笔卖出）
- 提供日期范围选择器，用户可调整查看范围
- 支持缩放、平移、悬停查看具体数据等交互功能

### 4. 买卖点标注

- 买入点：绿色向上箭头 ↑ 标记
- 卖出点：红色向下箭头 ↓ 标记
- 在卖出点旁边标注收益率百分比（基于 pnlcomm 计算）

### 5. 交易详情面板

- 图表下方显示该资产的所有交易记录表格
- 表格列包含：日期、类型（买入/卖出）、价格、仓位、手续费、pnl、pnlcomm
- 买入行用绿色背景高亮，卖出行用红色背景高亮

## 数据关联逻辑

1. `trade.csv` 中的 `asset` 字段对应 `prices.parquet` 中的 `bond_id`
2. `trade.csv` 中的 `date` 字段对应 `prices.parquet` 中的 `trade_date`
3. 每笔交易（买入或卖出）根据 `date` 定位到 K 线图上的具体位置

## 运行方式

```bash
# 安装依赖
pip install streamlit plotly polars

# 启动应用
streamlit run app.py
```

浏览器自动打开 http://localhost:8501 显示应用界面。

## 错误处理

- 文件路径无效或文件不存在时显示错误提示
- 文件格式不符合预期时显示格式要求说明
- 资产 ID 在价格数据中不存在时显示警告
- 单个资产没有完整交易记录（只有买入无卖出）时显示提示信息
